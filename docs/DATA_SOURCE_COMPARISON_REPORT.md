_# v1.6 펀더멘털 데이터 소스 추천 보고서

**Date**: 2024-11-27  
**Subject**: v1.6 Quality 엔진 개발을 위한 최적의 펀더멘털 데이터 소스 비교 분석 및 추천

---

## 📋 요약

v1.6 Quality 엔진 개발에 필요한 펀더멘털 데이터 소스로 **Alpha Vantage**, **Financial Modeling Prep (FMP)**, **Nasdaq Data Link (구 Quandl)** 세 가지를 비교 분석했습니다.

### 최종 추천

**🥇 주 소스: Financial Modeling Prep (FMP)**

**이유**:
1.  **가장 관대한 무료 티어**: 일 250회 API 호출 (Alpha Vantage의 10배)
2.  **합리적이고 투명한 가격**: 월 $19부터 시작하는 저렴한 유료 플랜
3.  **포괄적인 데이터**: 150개 이상의 엔드포인트, 전체 재무제표 및 비율 데이터 제공
4.  **확장성**: 무료 티어로 개발 시작 후, 필요시 저렴한 비용으로 프로덕션 전환 가능

### 개발 및 운영 로드맵

1.  **개발 단계**: **FMP 무료 티어** 사용 (일 250회)
    - 30개 종목의 펀더멘털 데이터를 매일 한 번씩 가져오기에 충분
2.  **프로덕션 단계**: **FMP Starter 플랜**으로 업그레이드 (월 $19)
    - 분당 300회 API 호출, 5년치 과거 데이터 제공
    - 실시간 트레이딩 및 백테스팅에 충분한 용량

### 차선책 및 미래 계획

-   **🥈 차선책**: **Alpha Vantage**
    -   FMP의 데이터 품질에 문제가 발생할 경우 백업으로 고려
    -   단, 일 25회 호출 제한은 개발에 매우 제약적
-   **🥉 미래 계획**: **Nasdaq Data Link**
    -   전략이 실제 수익성을 입증하고, 기관 수준의 데이터 품질이 필요하며, 예산이 허락될 경우 (연 $200-500) 최종적으로 이전 고려

## 2. 소스별 장단점 분석

### 🥇 Financial Modeling Prep (FMP)

**✅ 장점**:
-   **최고의 무료 티어**: 일 250회 호출은 v1.6 개발에 충분합니다.
-   **투명하고 합리적인 가격**: 월 $19의 Starter 플랜은 프로덕션 환경에 매우 경제적입니다.
-   **포괄적인 펀더멘털 데이터**: 150개 이상의 엔드포인트는 Quality, Value, Growth 등 다양한 팩터 개발에 필요한 모든 데이터를 제공합니다.
-   **쉬운 확장 경로**: 무료로 시작해서 필요에 따라 단계적으로 업그레이드하기 용이합니다.

**❌ 단점**:
-   일부 사용자로부터 데이터 정확성(반올림 오류) 및 데이터 누락에 대한 보고가 있습니다.
-   경쟁사 대비 역사가 짧습니다.

### 🥈 Alpha Vantage

**✅ 장점**:
-   API 문서가 매우 잘 되어 있고 사용하기 쉽습니다.
-   데이터 품질과 신뢰성이 좋다는 평이 많습니다.

**❌ 단점**:
-   **치명적인 무료 티어 제한**: 2024년부터 일 500회에서 **25회**로 대폭 축소되어, 30개 종목의 데이터를 한 번에 가져오는 것조차 불가능합니다.
-   프리미엄 플랜 가격이 비공개라 예산 책정이 어렵습니다.

### 🥉 Nasdaq Data Link (구 Quandl)

**✅ 장점**:
-   **최고 수준의 데이터 품질**: 기관 투자자들이 사용하는 데이터로, 정확성과 신뢰도가 매우 높습니다.
-   **Sharadar Core US Fundamentals** 데이터셋은 150개 이상의 지표와 24년치 과거 데이터를 제공하여 가장 포괄적입니다.

**❌ 단점**:
-   **높은 비용**: 유용한 펀더멘털 데이터는 대부분 유료이며, 개인 개발자가 사용하기에는 가격 장벽이 높습니다 (연 $200-500 이상).
-   무료 티어에서는 쓸만한 펀더멘털 데이터를 거의 찾을 수 없습니다.
-   API 사용법이 다른 두 소스에 비해 다소 복잡합니다.

## 3. 최종 추천 및 실행 계획

**결론적으로, Financial Modeling Prep (FMP)가 v1.6 개발의 주 소스로 가장 적합합니다.**

관대한 무료 티어는 초기 개발 비용을 없애주고, 합리적인 가격의 유료 플랜은 향후 프로덕션 환경으로의 전환을 원활하게 합니다. 데이터 커버리지 또한 Quality 엔진 개발에 필요한 모든 지표를 포함하고 있어 기능적으로도 부족함이 없습니다.

### 단계별 실행 계획

1.  **FMP API Key 발급**: FMP 웹사이트에서 무료 API 키를 발급받습니다.
2.  **데이터 수집 모듈 개발** (`utils/data_loader.py`):
    -   FMP API를 호출하여 30개 종목의 분기별/연간 재무제표 (손익계산서, 재무상태표, 현금흐름표) 및 주요 재무 비율 데이터를 가져오는 함수를 개발합니다.
    -   가져온 데이터를 `data/fundamental_data.csv` 파일로 저장하여 API 호출을 최소화합니다.
3.  **Quality 팩터 개발** (`utils/fundamental_factors.py`):
    -   저장된 펀더멘털 데이터를 바탕으로 ROE, 부채비율, 순이익률 등을 계산하여 **Quality Score**를 산출하는 함수를 개발합니다.
4.  **v1.6 엔진 통합**:
    -   기존 FV3c, ML9 엔진과 새로 개발한 Quality 엔진을 앙상블하여 v1.6 전략을 구성합니다.

데이터 품질에 대한 우려가 현실화될 경우, Alpha Vantage를 백업으로 사용하거나, 최종적으로 Nasdaq Data Link로의 이전을 고려하는 순차적 접근을 권장합니다.
