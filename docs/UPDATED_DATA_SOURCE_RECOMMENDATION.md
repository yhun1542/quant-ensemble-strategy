# v1.6 데이터 소스 최종 추천: 유료 계정 보유 시

**Date**: 2024-11-27  
**Subject**: Nasdaq Data Link 및 Alpha Vantage 유료 계정 보유를 전제로 한 v1.6 펀더멘털 데이터 소스 최종 추천

---

## 📋 요약: 추천이 완전히 변경되었습니다

**Nasdaq Data Link**와 **Alpha Vantage** 유료 계정을 이미 보유하고 계시다는 새로운 정보에 따라, 데이터 소스 추천을 전면 재검토했습니다.

### 최종 추천

**🥇 주 소스 (Primary): Nasdaq Data Link (Sharadar Core US Fundamentals 데이터셋)**

**🥈 보조 소스 (Secondary): Alpha Vantage**

**❌ 사용 안 함: Financial Modeling Prep (FMP)**

**핵심 결론**: 이미 **기관 수준(Institutional-grade)의 최고급 데이터**에 접근 권한이 있으므로, 무료 또는 저품질 데이터 소스를 고려할 이유가 전혀 없습니다. **Nasdaq Data Link를 주력으로 사용**하는 것이 v1.6 전략의 성공 가능성을 극대화하는 유일한 선택입니다.

---

## 1. 왜 Nasdaq Data Link가 최적인가?

Nasdaq Data Link (Sharadar 데이터셋)는 다른 두 옵션을 압도하는 명백한 우위를 가집니다.

| 항목 | Nasdaq Data Link (유료) | Alpha Vantage (유료) | FMP (무료) |
|:---|:---:|:---:|:---:|
| **데이터 품질** | ⭐⭐⭐⭐⭐ **기관 수준** | ⭐⭐⭐⭐ 좋음 | ⭐⭐⭐ 보통 |
| **펀더멘털 커버리지** | **150+ 지표** | 기본 재무제표 | 150+ 엔드포인트 |
| **과거 데이터 기간** | **24년** | 20년+ | 5년 |
| **신뢰성/정확성** | **최상** | 높음 | 보통 (오류 보고 있음) |
| **비용** | **이미 지불됨 (매몰 비용)** | **이미 지불됨 (매몰 비용)** | 무료 |

**결정적 이유**:

1.  **압도적인 데이터 품질**: Nasdaq Data Link의 데이터는 실제 헤지펀드와 기관 투자자들이 사용하는 데이터입니다. 데이터 클리닝에 소요되는 시간을 최소화하고, 백테스트의 신뢰도를 극대화할 수 있습니다.
2.  **가장 포괄적인 커버리지**: ROE, 부채비율, 각종 마진, 성장률 등 Quality 엔진 개발에 필요한 **모든 지표**를 포함한 150개 이상의 팩터를 제공합니다. 다른 소스는 일부 지표를 직접 계산해야 할 수 있습니다.
3.  **가장 긴 과거 데이터**: 24년치 데이터는 다양한 시장 국면(닷컴 버블, 2008 금융위기 등)에 대한 충분한 백테스팅을 가능하게 하여 전략의 견고성을 높여줍니다.
4.  **비용 효율성**: 이미 비용을 지불했기 때문에, 추가 비용 없이 최고 품질의 데이터를 사용할 수 있습니다. 이는 가장 비용 효율적인 선택입니다.

## 2. 소스별 역할 정의 및 실행 전략

### 🥇 주 소스: Nasdaq Data Link

**역할**: v1.6 Quality 엔진의 모든 펀더멘털 데이터 공급

**사용 데이터셋**: **Sharadar Core US Fundamentals (Table `SF1`)**

**주요 활용 지표**:

| 카테고리 | 주요 지표 (Nasdaq Data Link에서 직접 제공) |
|:---|:---|
| **수익성 (Profitability)** | `ROE`, `ROA`, `ROIC`, `NETMARGIN`, `OPMARGIN`, `GROSSMARGIN` |
| **재무 건전성 (Leverage)** | `DE`, `DEBT`, `ASSETS`, `EQUITY`, `INTCOV`, `CURRENTRATIO` |
| **가치 (Valuation)** | `PE`, `PB`, `PS`, `EVEBITDA` |
| **성장성 (Growth)** | `REVENUEGROWTH`, `EPSGROWTH` |
| **효율성 (Efficiency)** | `ASSETTURNOVER`, `INVTURNOVER` |

**실행 계획**:
1.  Nasdaq Data Link API를 사용하여 `SF1` 테이블에서 분기별(Quarterly) 데이터를 다운로드합니다.
2.  필요한 모든 지표(ROE, DE, NETMARGIN 등)를 추출하여 v1.6 Quality 엔진의 팩터로 사용합니다.
3.  24년치 데이터를 활용하여 장기간 백테스트를 수행합니다.

### 🥈 보조 소스: Alpha Vantage

**역할**: 백업, 실시간 데이터, 데이터 검증

**실행 계획**:
1.  **데이터 검증**: Nasdaq Data Link에서 가져온 데이터 일부를 Alpha Vantage 데이터와 교차 확인하여 이상치를 탐지합니다.
2.  **실시간 데이터**: 만약 v1.6 전략을 실시간 트레이딩에 사용할 경우, Alpha Vantage의 실시간 가격 API를 보조적으로 활용할 수 있습니다.
3.  **커버리지 보완**: Nasdaq Data Link에 누락된 특정 종목이 있을 경우, Alpha Vantage 데이터로 보완합니다.

### ❌ 사용 안 함: Financial Modeling Prep (FMP)

**결론**: **사용할 이유가 없습니다.**

이미 두 개의 우수한 유료 데이터 소스를 보유하고 있으므로, 상대적으로 품질이 낮고 API 호출 제한이 있는 무료 소스를 사용할 필요가 없습니다. 데이터 소스를 여러 개 혼용하면 데이터 불일치 문제만 발생할 수 있습니다.

## 3. 최종 권장사항

**v1.6 엔진의 성공은 데이터 품질에 달려있습니다.**

이미 보유하고 계신 **Nasdaq Data Link**라는 강력한 무기를 사용하지 않을 이유가 없습니다. FMP의 무료 티어나 Alpha Vantage의 편의성에 안주하지 말고, 처음부터 최고 품질의 데이터로 전략을 구축하는 것이 장기적으로 시간과 노력을 아끼는 길입니다.

**즉시 실행할 것**:
1.  **Nasdaq Data Link API 연동**: `SF1` 데이터셋 접근을 최우선으로 개발합니다.
2.  **데이터 수집 스크립트 작성**: 30개 종목에 대한 24년치 펀더멘털 데이터를 로컬에 저장하는 스크립트를 구현합니다.
3.  **Quality 팩터 개발**: 저장된 데이터를 기반으로 ROE, 부채비율, 순이익률 등을 조합하여 Quality Score를 계산하는 로직을 개발합니다.

이 접근법을 통해 v1.6 엔진의 신뢰성과 성능을 극대화할 수 있습니다.
